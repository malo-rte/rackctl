= Eleven Rack ALSA driver (native UAC2 soundcard for the Avid Eleven Rack)

A `snd-usb-audio` patch that gives the Avid/Digidesign Eleven Rack (`0dba:b011`)
a native ALSA soundcard: *8 capture channels / 6 playback channels, 24-bit in
`S32_LE` slots, 44.1 / 48 / 88.2 / 96 kHz*, full-duplex, plus its mixer and MIDI.

Background and the full USB analysis:
`../../docs/eleven-rack-audio-driver-scope.adoc`.

Base kernel: *Linux 6.18*.

== Status: WORKING, lab-validated (2026-07-16)

Confirmed on a real unit (kernel 6.18.38): capture and playback both stream real
audio, full-duplex, at 48 kHz and 96 kHz with *zero xruns*; `arecord -l` / `aplay
-l` list `card: Rack [Eleven Rack]`. Capture is clock master, so the exact
byte counts (96 kHz capture -> exactly 96000x8x4 bytes/s) prove the device is
genuinely clocking at the requested rate. The mixer and MIDI work in parallel.

Validated with lab instruments (Rigol signal generator on the guitar input, Rigol
scope on the headphone output, over raw-TCP SCPI): capture is pitch-accurate
(1 kHz in -> ~1000 Hz on ch1/ch3), and playback reads on the scope as a clean
1 kHz sine (freq 999 Hz sd 7, Vpp 464 mV sd 5, no dropouts). See the "Lab
validation" section in `../../docs/eleven-rack-audio-driver-scope.adoc` for the
full channel map and the resolution of the earlier VM-passthrough red herrings.

*Userspace:* the kernel driver is validated; the PipeWire experience still needs
tuning. `wireplumber-eleven-rack.{conf,nix}` here is a baseline (no node suspend,
Main as FL/FR + discrete AUX, documented channel map).

== The device's three non-conformances, and the fix

The Eleven Rack is standard USB Audio Class 2.0, but deviates in three ways that
each needed a fix. The patch (`eleven-rack-uac2-quirk.patch`) touches five files:

[cols="1,4",options="header"]
|===
| File | Why
| `sound/usb/quirks-table.h` | The audio interfaces are marked *vendor-class* (`0xFF`) so the stock driver skips them. A composite quirk forces standard UAC2 parsing: `IGNORE(0)` (DFU), `STANDARD_MIXER(1)`, `STANDARD_MIDI(2)`, `STANDARD_AUDIO(3)` playback, `STANDARD_AUDIO(4)` capture.
| `sound/usb/card.c` | The *DFU interface (0) enumerates first* and became `chip->ctrl_intf`, so terminal/clock lookups for the streaming interfaces searched an audio-less interface and failed ("bogus bTerminalLink"). Fix: pick a real AudioControl interface for `chip->ctrl_intf`.
| `sound/usb/format.c` | The firmware *does not answer the UAC2 SAM_FREQ RANGE* request, so standard rate enumeration finds no rates -> no PCM. Fix: a fixed-rate-list fallback (44.1/48/88.2/96 kHz) for the device.
| `sound/usb/pcm.c` | The endpoints declare a *read-only PITCH control that the firmware NAKs on enable*, which was fatal. Fix: under `QUIRK_FLAG_IGNORE_CTL_ERROR`, treat a failed PITCH enable as non-fatal.
| `sound/usb/quirks.c` | `QUIRK_FLAG_IGNORE_CTL_ERROR` for the device (covers the read-only pitch and the mixer-unit min/max the firmware does not answer).
|===

Implicit feedback (playback slaved to capture) is set up automatically by standard
parsing from the CS endpoint descriptors -- no manual sync wiring needed.

== Persistent install (NixOS)

`nixos-eleven-rack.nix` applies the patch via `boot.kernelPatches` on Linux 6.18:

[source,nix]
----
imports = [ ./contrib/eleven-rack-alsa-quirk/nixos-eleven-rack.nix ];
----

`nixos-rebuild switch` rebuilds the kernel (slow, once) and the Rack comes up as a
soundcard after reboot. On other distros, apply `eleven-rack-uac2-quirk.patch`
(`git apply` / `patch -p1`) to the 6.18 source and rebuild `snd-usb-audio`.

== Fast iteration (module-only rebuild + hot reload)

For developing the patch without a full kernel rebuild, build just
`snd-usb-audio.ko` against the stock kernel's build tree and hot-swap it (see the
NixOS test-VM setup in `nixos-cfg/hosts/vm/eleven-rack/`: `module-only.nix` builds
the module via `nix build .#eleven-snd-usb`, `eleven-mod.sh` rebuilds + reloads it
with no reboot). The general recipe on any distro with the matching kernel source:

[source,sh]
----
# patched sound/usb copied to $M, kernel build tree at $KBUILD
make -C "$KBUILD" M="$M/sound/usb" modules
sudo rmmod snd_usb_audio && sudo insmod "$M/sound/usb/snd-usb-audio.ko"
----

== Verify

[source,sh]
----
arecord -l ; aplay -l ; cat /proc/asound/cards            # expect "Rack [Eleven Rack]"
cat /proc/asound/card1/stream0                            # 6-out / 8-in, 4 rates, 24-bit
arecord -D hw:1,0 -f S32_LE -c 8 -r 96000 -d 2 /tmp/r.wav # 8-ch capture at 96 kHz
speaker-test -D hw:1,0 -c 6 -r 48000 -F S32_LE -l 1       # playback
----

== Known limitations (not blockers)

* The 8 "Analog In Volume" mixer controls (mixer unit `0x40`) read as
  non-functional -- the firmware does not answer their min/max request. The
  main volume/switch controls work; input gain is also settable on the unit /
  over MIDI. `IGNORE_CTL_ERROR` silences the errors.
* Channel names are generic (FL/FR/...) rather than labelled guitar/dry/reamp/
  USB-return -- cosmetic routing, addable later from the terminal descriptors.

== Upstreaming

The patch is structured for `alsa-devel` submission. Before sending, generalise
the `format.c` fallback (it currently piggybacks the `line6_...` rate-quirk helper)
and add your `Signed-off-by`.
