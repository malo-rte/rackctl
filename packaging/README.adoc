= Packaging and desktop integration

Distribution assets for the Tascam US-16x08 mixer tools and notes on installing
them by hand. The Nix flake (`../flake.nix`) wires all of this up automatically;
the steps here are for packagers and for a manual install.

== Binaries

Build the release binaries and copy them onto the `PATH`:

[source,bash]
----
cargo build --release --workspace
sudo install -Dm755 target/release/rackctl-us16x08    /usr/bin/rackctl-us16x08
sudo install -Dm755 target/release/rackctl-us16x08-gui  /usr/bin/rackctl-us16x08-gui
----

== Desktop entry

`rackctl-us16x08-gui.desktop` launches the GUI from a desktop menu or launcher. Its
`StartupWMClass` matches the GUI's Wayland `app_id` (`rackctl-us16x08-gui`), so
compositors associate the window with the entry.

[source,bash]
----
sudo install -Dm644 packaging/rackctl-us16x08-gui.desktop \
  /usr/share/applications/rackctl-us16x08-gui.desktop
----

The entry references an icon named `rackctl-us16x08-gui`. Until an icon ships, launchers
fall back to a generic one; installing a `rackctl-us16x08-gui.png`/`.svg` into the
hicolor icon theme (for example
`/usr/share/icons/hicolor/scalable/apps/rackctl-us16x08-gui.svg`) makes it appear.

== Shell completions

`rackctl-us16x08` generates its own completion scripts, so they always match the
current commands and flags. Generate and install them for the shells you want:

[source,bash]
----
# bash
rackctl-us16x08 completions bash | sudo tee /usr/share/bash-completion/completions/rackctl-us16x08 >/dev/null

# zsh  (a directory on your $fpath)
rackctl-us16x08 completions zsh  | sudo tee /usr/share/zsh/site-functions/_rackctl-us16x08 >/dev/null

# fish
rackctl-us16x08 completions fish > ~/.config/fish/completions/rackctl-us16x08.fish
----

Supported shells: `bash`, `zsh`, `fish`, `elvish`, `powershell`.
